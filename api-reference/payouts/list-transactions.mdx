---
title: 'List Transactions'
description: 'Retrieve a paginated list of your transactions'
---

The List Transactions endpoint returns a paginated list of all transactions in your account, including both collections (payins) and disbursements (payouts).

## Endpoint

```
GET /v1/transactions
```

## Query Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `perPage` | integer | No | 10 | Number of transactions per page (min: 10, max: 50) |

## Example Request

<CodeGroup>
```bash cURL
curl -X GET "https://api.useglyde.co/v1/transactions?perPage=25" \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json"
```

```javascript Node.js
const axios = require('axios');

const response = await axios.get('https://api.useglyde.co/v1/transactions', {
  params: {
    perPage: 25
  },
  headers: {
    'Authorization': 'Bearer sk_live_your_secret_key',
    'Content-Type': 'application/json'
  }
});

console.log(response.data);
```

```python Python
import requests

params = {
    'perPage': 25
}

headers = {
    'Authorization': 'Bearer sk_live_your_secret_key',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://api.useglyde.co/v1/transactions',
    params=params,
    headers=headers
)

print(response.json())
```

```php PHP
<?php

curl_setopt_array($curl, [
    CURLOPT_URL => 'https://api.useglyde.co/v1/transactions?perPage=25',
```

## Response Fields

### Transaction Object

| Field | Type | Description |
|-------|------|-------------|
| `reference` | string | Glyde's unique reference for the transaction |
| `merchant_reference` | string | Your provided reference |
| `type` | string | Transaction type: `credit` (collection) or `debit` (transfer) |
| `amount` | integer | Transaction amount in smallest currency unit |
| `fee` | integer | Fee charged for the transaction |
| `status` | string | Transaction status: `pending`, `successful`, or `failed` |
| `created_at` | string | Timestamp when the transaction was created |

### Pagination Object

| Field | Type | Description |
|-------|------|-------------|
| `current_page` | integer | Current page number |
| `per_page` | integer | Number of items per page |
| `total` | integer | Total number of transactions |
| `total_pages` | integer | Total number of pages |

## Transaction Types

| Type | Description |
|------|-------------|
| `credit` | Money received (collections/payins) |
| `debit` | Money sent (transfers/payouts) |

## Filtering Tips

To filter transactions effectively:

1. Use pagination to browse through large datasets
2. Fetch individual transaction details using the [Fetch Transaction](/api-reference/payouts/fetch-transaction) endpoint
3. Use your `merchant_reference` to track specific transactions in your system

<Info>
  For real-time transaction updates, set up [webhooks](/api-reference/webhooks) instead of polling this endpoint.
</Info>
