Settlements represent the batched transfer of funds from successful collections to your bank account. Glyde automatically settles funds based on your settlement schedule.

## How Settlements Work

<Steps>
  <Step title="Collect Payments">
    Receive payments from customers through collections
  </Step>
  <Step title="Funds Accumulate">
    Successful payments are held in your Glyde wallet
  </Step>
  <Step title="Settlement Initiated">
    At your settlement period, funds are batched together
  </Step>
  <Step title="Bank Transfer">
    The total amount (minus fees) is transferred to your bank account
  </Step>
</Steps>

## List All Settlements

Retrieve a paginated list of your settlements.

### Endpoint

```
GET /v1/settlement
```

### Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `per_page` | integer | Yes | Number of records per page |

### Example Request

<CodeGroup>
```bash cURL
curl -X GET "https://api.useglyde.co/v1/settlement?per_page=20" \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json"
```

```javascript Node.js
const axios = require('axios');

const response = await axios.get('https://api.useglyde.co/v1/settlement', {
  params: {
    per_page: 20
  },
  headers: {
    'Authorization': 'Bearer sk_live_your_secret_key',
    'Content-Type': 'application/json'
  }
});

console.log(response.data);
```

```python Python
import requests

params = {
    'per_page': 20
}

headers = {
    'Authorization': 'Bearer sk_live_your_secret_key',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://api.useglyde.co/v1/settlement',
    params=params,
    headers=headers
)

print(response.json())
```

## Get Settlement Details

Retrieve detailed information about a specific settlement including the transactions included.

### Endpoint

```
GET /v1/settlement/{id}
```

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string | Yes | The settlement ID |

### Example Request

```bash
curl -X GET "https://api.useglyde.co/v1/settlement/stl_abc123" \
  -H "Authorization: Bearer sk_live_your_secret_key" \
  -H "Content-Type: application/json"
```

## Settlement Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique settlement identifier |
| `amount` | integer | Total gross amount of all transactions |
| `fee` | integer | Total fees deducted |
| `net_amount` | integer | Amount transferred to your bank (amount - fee) |
| `status` | string | Settlement status: `pending`, `processing`, `completed`, `failed` |
| `transactions_count` | integer | Number of transactions in this settlement |
| `settled_at` | string | When the settlement was completed |
| `created_at` | string | When the settlement was created |

## Settlement Statuses

| Status | Description |
|--------|-------------|
| `pending` | Settlement is queued and waiting to be processed |
| `processing` | Settlement is being transferred to your bank |
| `completed` | Settlement has been successfully transferred |
| `failed` | Settlement failed (will be retried) |

<Info>
  Settlement schedules vary based on your account type and configuration. Contact support if you need to adjust your settlement schedule.
</Info>
